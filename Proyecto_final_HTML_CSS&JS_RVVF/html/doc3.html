Conceptos importantes antes del manejo de Github

Antes de comenzar a revisar los comandos básicos de Github es importante tener presentes algunos conceptos básicos como el saber qué es un repositorio en línea o remoto, un repositorio local y un repositorio remoto . Es importante mencionar que al comenzar a utilizar github de forma colaborativa usualmente se nos pedirá que hagamos un fork al repositorio de un compañero o de la empresa en la que estemos trabajando y una vez hecha nuestra parte del trabajo se tendrá que relizar un pull request al compañero o la empresa, a continuación se explicarán todos estos términos con el fin de que la próxima vez que nos enfrentemos a esto sepamos que se debe hacer .

Repositorio
Es un espacio de almacenamiento en donde se guardan y acceden a archivos como por ejemplo algún proyecen el que se esté trabajando.
Para más información de click aquí

Repositorio en línea
Repositorio que está guardado en el servidor de github.

Repositorio local
Se trata de una copia de algún repositorio en línea dentro de nuestro ordenador.

Repositorio o remoto
Se trata del repositorio en línea de un tercero al cual hicimos un fork.
Para más información de click aquí

Fork
Crear una copia del repositorio en línea de otro perfil en nuestro perfil de github. Desde este repositorio, podemos colaborar y sugerir cambios en el repositorio original.

Pull request
Se trata de una solicitud de incluir cambios en el repositorio de un tercero realizados en un repositorio en línea surgido a partir de realizarle un fork.

Proceso para actualizar nuestro repositorio remoto una vez hecho el Fork de algún otro compañero

  • Una vez que tenemos la "copia" en nuestro repositorio remoto, se debe primero clonar a nuestro repositorio local

    • $git clone url

    donde la url es de nuestro repositorio remoto.

  • Teniendo la copia, dentro de nuetro repositorio local nos movemos a la carpeta del repositorio
  • Una vez ahí, actualizamos nuestro repositorio local desde el repositorio original pidiendo los cambios que se hayan hecho:

    • $git remote add --track master NICKNAME url

    donde el NICKNAME un identificador que le daremos al repositorio original y éste puede ser cualquier nombre, el url es del repositorio remoto original.

    Ten en cuenta que este proceso es cuando nosotros no somos los propietarios del repositio original. En dado caso de que quisieramos actulizar nuestro repositorio siendo nosotros los propietarios el comando anterior es innecesario y simplemente tendríamos que escribir en la línea de comando los siguiente:

    $git pull origin RAMA

  • Si modificamos alguno de los acrhivos dentro de nuestro repositorio local, debemos agregarlos y comentarlos

    • $git add README.md
    $git commit -m "Ejemplo"

  • De lo contrario, nos movemos a la rama que queremos actualizar y "bajamos" los cambios con el comando pull

    • $git pull NICKNAME rama

    donde la url es de nuestro repositorio remoto.

  • Finalmente actualizamos la respectiva rama en nuestro repositorio remoto

    • $git push origin RAMA
